Stage 0: What the Heck Is Going On?

Intro

The web basically consists of four general elements: browsers (with their usually human users), hypertext markup language (HTML), hypertext transfer protocol (HTTP), and servers.

Browsers

Chrome, Firefox, Internet Explorer, etc. Everyone knows what these are! They're how users access and interact with html docs.

HTML

This stands for Hypertext Markup Language, and it is the basis for the type of documents most often encountered on the web.

HTTP

Which stands for Hypertext Transfer Protocol. HTTP is the protocol that allows communication between servers and browsers, so that servers know which documents to send to browsers.

Servers

Servers are just computers that are optimized not for interaction with users but rather for storage.

Learning

Learning is a continuum from novice to expertise. Four stages are: ignorance, awareness, ability, and fluency. Ability to fluency is the hardest, longest stage. We're shooting for Ability in some topics and Awareness in many others. Frustration is inevitable; you should always ask for help, take a break, and/or just keep going.

Document Object Model

Document Object Model (DOM) is the tree-like structure of a page. "a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages. The nodes of every document are organized in a tree structure, called the DOM tree."

Boxes and Boxifying

Everything front-end is boxes, so it's efficient to think of the page that way. Boxify is a word for when you take someone's mockup/pdf of a page and determine the number/placement of boxes you'd need for the code.

CSS is used for style, so the ex is if you're moving, html = the boxes you put your stuff in and css = the instructions you write on them ('goes in kitchen' or 'handle with care').

HTML, CSS, and JavaScript

Websites qua house: html is structure (walls, rooms), css are style of house (carpeting color, wall colors), javascript are interactive components (tv remote, garage door opener). Webtools too (screwdriver).

Chrome Developer Tools

Chrome->tools->dev tools (wikipedia), shows not code itself but rather structure of page as browser interprets the code; you can get down to the code, though, including plain text.

Changing stuff in dev mode is displayed then in the lefthand panel to show you what would change there. Style of html stuff is shown below in another window.

HTML ->language-> syntax and rules; each tag is like a word in html language, or they're elements in tree. tags can have attributes (class, href, etc.);

CSS (Cascading Style Sheets) allows you to use syntax and rules to change look of page; in CSS, unchecking "border radius 50%" changes circles to squares.

CSS and Style Modeling

You must link from html doc to css in order to apply style to your page--in this example it was making the description part of your page red.

Cascading Style Sheets: imagine like national, state, municipal, house laws. But here, the most specific rule (house rule, here) gets applied to every element--it cascades up to national, that is.

Selector = what a style on page gets applied to. Font can be normal, italic, and oblique. Font-weight light(er) or bold(er).

Best Practices for Coding Style

Consistent indentation levels is crucial! The pre tag preserves any formatting present within its elements. Use double quotes. Use comments to clarify where a big div ends, for example, but in general use comments sparingly. The structure of your code should make your intentions obvious and comments unnecessary. Try to find the shortest link for a URL. In CSS, keep one attribute and one value per line, and keep attributes indented to same level.

Boxes are stylistically made up of: content, padding, border, and margins. By default, the width of a box is the sum of the content, padding, and border values.

Flex boxes control positioning of boxes: add flex to value of size attribute.

Code, Test, Refine

  1. Look for natural boxes;
  2. Look for repeated styles and semantic elements;
  3. Write your HTML;
  4. Apply styles (from biggest to smallest);
  5. Fix things!
